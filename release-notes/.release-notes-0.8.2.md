# Release v0.8.2 - CI/Release Automation, Storage & Version sync fixes

## What's New

### Added
- **GitHub Actions**: Neue Workflows für `dev` und `release` (`dev-push.yml`, `release.yml`) zur automatisierten Version-/SHA-Setzung und Release-Verwaltung.
- **RELEASE_GUIDE.md**: Detaillierte Anleitung zum Release-Prozess wurde hinzugefügt.
- **User-scoped storage**: LocalStorage-Schlüssel sind jetzt auf Nutzer scopes umgestellt; `AuthProvider.logout()` leitet jetzt zu `/login` weiter.
- **Admin / Ops**: Verbesserungen an Admin-UI und System‑Metriken / Version-Anzeige im Admin-Panel.

### Changed
- Entfernt veraltete Workflows (`docker-publish.yml`, `sync-version.yml`, `version-release.yml`).
- CI/Build: Dev-Workflow setzt `version='dev'` und `BUILD_SHA`/`BUILD_VERSION` werden bevorzugt als Build‑Variablen. Workflows/Trigger für dev/main wurden angepasst.

### Fixed
- **Backend version & SHA sync**: Kopiere `backend/package.json` ins Docker-Image; Version‑Mismatch‑Check wurde auf SHA‑Vergleich umgestellt, damit Admin‑Panel die korrekte Build‑Information anzeigt.
- **Docker / Vite**: Korrekte Übergabe von `BUILD_SHA` / `BUILD_VERSION` in Dockerfile und Vite‑Defines, damit Versionen konsistent im Frontend/Admin sichtbar sind.
- **package-lock.json**: Lockfiles wurden korrekt in Version‑Commits aufgenommen.
- **Admin dev package version**: Dev‑Version im Admin wurde normalisiert (Anzeige/Formatierung korrigiert).
