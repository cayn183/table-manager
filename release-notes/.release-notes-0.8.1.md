# Release v0.8.1 - Verbesserte Synchronisation und Timeline

## What's New

### Added
- **Unload / keepalive sync**: Best‑effort Batch‑Endpoint (`POST /events/batch`) und `syncUserDataOnUnload()` im Frontend verwenden `fetch(..., { keepalive: true })` beim Verlassen/Reload der Seite.
- **Hydration on login**: Beim Login werden serverseitige Events/Räume in user‑scoped LocalStorage geschrieben, damit andere Browser/Devices sofort die Daten sehen.

### Changed
- **Synchronisation**: `syncUserData()` hat jetzt Retry/Exponential‑Backoff (3 Versuche) und wirft Fehler nach wiederholtem Fehlschlag, damit manuelle Saves Fehler anzeigen können.
- **Timeline**: Timeline‑View nutzt nun höhenbasierte Spalten‑Logik mit Fortsetzungs‑Headern; lange Slots werden segmentiert und sauber über Spalten verteilt.
- **Autosave / save flow**: Manuelle Speichern wartet nun auf `syncUserData()` und zeigt Saving‑Indikatoren; Autosave überspringt wenn `isDirty===false`.

### Fixed
- **API upsert/delete**: Backend `/events` akzeptiert jetzt client‑id upsert (falls id existiert, update/verify owner), und `DELETE /events/:id` wurde hinzugefügt.
- **TypeScript/Build**: Typendefinitionen (`vite-env.d.ts`) und `confirmSaveRoom` async‑Fixes entfernt TypeScript‑Fehler, die Docker‑Builds zum Abbruch brachten.
- **Timeline header icon**: Uhrsymbol wird in Fortsetzungen nicht mehr wiederholt.

## Technical Details
- Version: `0.8.1`
- Release Date: `2026-01-29`
- Highlights: Stabilere Cross‑device‑Sync (hydration+unload), robustere client‑side sync mit retries, verbesserte Timeline Spaltenaufteilung und UI‑Polish für Saves und Autosave.
