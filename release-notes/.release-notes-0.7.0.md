# Release v0.7.0 - Verbesserte Druckansicht & Platzierungsrobustheit

## What's New

### Added
- Druckansicht: Dritte "Bemerkung"-Zeile in Listen und auf Kartenlabels zur besseren Anzeige von Hinweisen

### Changed
- Bemerkungen werden jetzt durch ein kleines "⚠️" angezeigt und vor den Familiennamen gesetzt (sichtbar in Listen und Karten)
- Icons in Druck-Labels wurden verkleinert; Abstand und Zeilenhöhe für Drucketiketten wurden reduziert; Tischnummer wurde für Druckansicht etwas vergrößert
- Maximale Länge für `note` (Bemerkung) erhöht auf 50 Zeichen beim Import und in Formularen

### Fixed
- Vermeidet Überschneidungen bei Einzelzuweisungen über Kontextmenü: Verwendung von `tryPlaceOnTable` zur Bestimmung gültiger x/y/rotation-Werte
- Rotation/Placement: Erzeugung rotierter Layout-Varianten verbessert, so dass Gruppen der Größe 6/8 zuverlässig passende Rotationen finden können

## Technical Details

- Version: `0.7.0`
- Release Date: `2026-01-23`
- Affected files (Beispiele): `src/components/PrintViewPage.tsx`, `src/components/Room.tsx`, `src/components/Importer.tsx`, `src/utils/layoutUtils.ts`, `src/styles.css`

## Installation / Release Steps

1. Merge `dev` into `main` and push the merge commit.
2. Tag the release and push the tag:

```bash
git checkout main
git merge --no-ff dev -m "chore(release): 0.7.0"
git tag -a v0.7.0 -m "Release 0.7.0"
git push origin main
git push origin v0.7.0
```

The repository contains a workflow that will create the GitHub Release when a `v*` tag is pushed; see `.github/workflows/version-release.yml`.
